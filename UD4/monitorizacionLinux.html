Monitorización de Linux

La monitorización de Linux puede hacerse de muchas formas, pero la mas común es mediante comandos y la consulta de los archivos de configuración y log( en Linux todo son archivos de texto ). Si la consulta es puntual lo normal es hacer una conexión SSH y comprobar ciertos parámetros, pero lo mas normal es crear scripts que nos permitan automatizar las tareas periódicas.

Archivos de configuración

Linux se configura por medio de archivos de texto. Algunos de los mas importantes son:

  • /etc/host: Contiene resoluciones DNS estáticas y únicas del usuario. Como mínimo contiene una entrada para localhost.
  • /etc/resolv.conf: Contiene las direcciones IP de los servidores DNS. Se actualiza automáticamente si se configura la interfaz como DHCP o puede que contenga 127.0.0.1. En todo caso solo debería tocarse para configurar manualmente nuestro servidor DNS ( aunque eso también podemos hacerlo desde el archivo de interfaces )
  • /etc/network/interfaces: se utiliza para definir los nombres lógicos de las interfaces de redes, a los que se les asociará una configuración determinada. El comando ifup es el encargado de relacionar una interfaz física con una interfaz lógica.
  • /etc/passwd: contiene una linea por cada usuario del sistema indicando a que grupos.

Archivos de log

  • /etc/proc Contiene directorios y archivos de log del hardware del sistema.
  • /var/log/auth.log: contiene los accesos al sistema, incluidos los intentos fallidos.
  • /var/log/syslog: se encarga de registrar los mensajes de seguridad del sistema.
  • /var/log/cron.log: registro del demonio cron
  • /var/log/dpkg.log: registro de actuación sobre los paquetes del sistema

Comandos Linux

  • HARDWARE
    • lshw Información de todo el hardware
    • lscpu Información de la CPU
    • lspci Información acerca de las ranuras PCI en el sistema y los dispositivos conectados a éstas
    • dsmeg Información de todo el hardware detectado por el sistema
    • fdisk -l Información sobre los discos.
    • lsblk Información sobre los dispositivos de bloque ( particiones, CD-ROM, volumenes vlm, etc.)
    • df: muestra el espacio libre del sistema de archivos. [df -h /]
    • free: proporciona información sobre el uso de memoria fisica y de swap.
  • RED
    • ifconfig: Muestra la configuración de las interfaces de red configuradas. Con la opción -a muestra también las no configuradas.
    • ifup Enciende ( levanta ) una interfaz de red.
    • ifdown Apaga ( baja ) una interfaz de red.
    • netstat: muestra información sobre los puertos.
    • arp: muestra información sobre las tablas ARP.
    • iftop: si nuestro servidor actua como proxy podemos ver las estadísticas de conexión de cada IP.
  • SISTEMA
    • uname: Información básica del sistema.
    • last: muestra un listado con los ultimos usuarios que han entrado al sistema.
    • find: busca archivos o directorios en el arbol de directorios.
  • PROCESOS
    • top: lista de procesos ordenados por uso de CPU
    • ps: pid de los procesos. Con la opción -aux obtenemos todos ordenados por numero de PID.
    • kill : mata el proceso identificado por su PID.
  • MISCELANEA
    • wc: cuenta las lineas( con la opción -l) de la salida de un comando o un archivo.
    • time: tiempo de ejecución de un comando.
    • who: usuarios logueados en el sistema
